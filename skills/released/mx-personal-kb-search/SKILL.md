# 妙想私域知识库检索

通过**自然语言问句**对用户的私域知识库进行语义检索，返回结构化的 Markdown 检索结果，适用场景包括：
- **基于私域资料回答个性化问题（研报、公告、财报、自定义文档等）**
- **核实知识库中是否已沉淀某类资料（人物、公司、主题、政策等）**
- **将检索到的证据片段交给上层 Agent，用于综合答复或溯源**
- **用户泛化问法的"先检索后回答"路径**

## 功能范围

### 基础检索能力
- 输入自然语言问句，调用妙想私域知识库检索接口，返回最相关的若干内容片段
- 每次仅处理**一次**检索查询，不做多轮串联或上下文记忆
- 返回可读 Markdown 检索结果（来源文件、类型、时间、原文片段、预览链接）
- 支持将结果保存为本地 `.md` 文件，便于后续复用与追溯
- 兼容接口的业务状态返回（如未开通、空库、未检索到内容），**原样**透传给上层

### 触发规则（何时使用本技能）

妙想产品里有两个完全不同的存储概念，**务必区分**：

- **工作空间** —— 妙想 claw 会话内上传的文件，体量小、临时使用
- **私域知识库** —— 独立的知识库产品，存放用户长期沉淀的资料，本技能服务于此

**判断要点：以"知识库"三个字作为硬触发信号。** 用户没说"知识库"，哪怕说了"我上传的"、"我的资料"、"我导入的"、"我的文档"、"我刚传的那份"，也大概率是在指工作空间——不要触发本技能。

若上下文中已明确"知识库"实体，用户后续使用"它/那份/这份"等代词继续提问，也应触发；但需先做指代消解再传入 `--query`，脚本内部不会读取对话上文。

### 不触发规则（何时不要使用本技能）
- 用户问句中**未出现"知识库"**字样，仅有"我上传的"、"我的资料"、"我导入的"等模糊表达（大概率指向工作空间）
- 用户咨询的是公开行情、个股诊断、基金诊断等与私域资料无关的问题
- 用户要求"上传 / 删除 / 管理"知识库文件（属产品端写操作，不在本技能范围）
- 用户要求多轮检索串联、构建检索增强问答管线等高级编排（应由上层 Agent 编排）

如果用户问句指代不明（可能是知识库也可能是工作空间），**优先反问**"是要查您私域知识库里的内容，还是当前工作空间里的文件"，而不是擅自触发。

### 触发示例

| 触发（明确指向知识库） | 不触发（指向工作空间或模糊） |
|---|---|
| 我的知识库里有没有关于宁德时代的研报？ | 我刚上传的那份研报里说了什么？ |
| 私域知识库里这只股票的最新观点是什么？ | 基于我导入的资料分析这家公司 |
| 基于我的知识库分析一下这家公司 | 我的那份财报怎么说？ |
| 翻一下知识库里那份财报 | 今天大盘怎么样？ |
| 知识库中有没有提到这次政策的解读？ | 把这只股票净值导出为 CSV |

## 快速开始

### 1. 命令行调用

```bash
python3 {baseDir}/scripts/get_data.py --query "宁德时代最近的观点"
```

**输出示例**
```text
Saved: /path/to/workspace/kb_search/kb_search_90bf169c.md
（随后输出 Markdown 检索结果）
```

**参数说明：**

| 参数 | 说明 | 必填 |
|---|---|---|
| `--query` | 用户原始自然语言问句（含上下文指代请先消解后再传入） | ✅（`--query` 或 stdin 二选一） |
| `--no-save` | 仅输出结果，不写入本地文件 | 否 |

### 2. 代码调用

```python
import asyncio
from pathlib import Path
from scripts.get_data import search_personal_kb

async def main():
    result = await search_personal_kb(
        query="宁德时代最近的观点",
        output_dir=Path("workspace/kb_search"),
        save_to_file=True,
    )
    if "error" in result:
        print(result["error"])
    else:
        print(result["content"])
        if result.get("output_path"):
            print("已保存至:", result["output_path"])

asyncio.run(main())
```

## 输出规范

通过脚本拿到检索结果后，**对用户的可见回复必须以接口返回的内容为依据**，避免编造来源、虚构片段或冒充检索结果。仅当脚本明确返回 `error`、或正文为空时，才用简短文字说明失败原因；**禁止**在失败时杜撰检索内容。

当接口返回业务状态话术时，按下表处理：

| 脚本输出 | 你应该做的 |
|---|---|
| `您暂无知识库权限，请联系客服开通投研助理产品权限，客服电话400-620-1818转2-B端` | 说明用户尚未开通权限，**原样**转给用户。客服电话是兜底联系方式，**不要省略或改写**。 |
| `您的知识库暂无文件，请前往妙想知识库上传吧` | 说明知识库尚未上传任何文件，**原样**转给用户，可补一句友好提示但话术主体保持不变。 |
| `未在您的知识库中检索到与该问题相关的内容，请换种说法再试` | **原样**转给用户，可建议尝试更换关键词或换一种问法。 |
| `知识库状态检查失败：…` / `知识库检索失败：…` / `内部错误：…` | 告知用户当前服务异常，建议稍后重试。**不要**用通用知识凑出答案。 |
| 检索到的 Markdown 正文（含多条片段） | 以接口正文为主体回答，可基于片段做轻度总结，但不要捏造未出现的字段或来源。 |

## 常见问题


**为什么需要保持原始问句？**  
→ 检索接口依赖用户自然语言语义，建议避免改写导致检索偏移；如含指代代词，请在调用前先做指代消解。

**如何只看输出，不落盘？**
```bash
python3 -m scripts.get_data --query "宁德时代最近的观点" --no-save
```

**脚本返回了空内容怎么办？**  
→ 如果 `raw` 字段中存在业务状态话术（如"暂无文件"），按"输出规范"表处理；否则视作服务异常并提示稍后重试，不要编造检索结果。

## 合规说明

- 检索结果仅供参考，不构成投资建议，输出时应附必要的风险提示。
- 检索是"读"操作，本技能不涉及上传 / 删除 / 修改知识库文件。
- 不替脚本判断权限：若返回"暂无权限"等话术，不要用通用知识冒充检索结果。
- 禁止在代码或提示词中硬编码账号 ID、会话 ID 或 token。
- 环境变量按敏感信息处理，不在日志或回复中泄露。
- 接口失败时不得编造结论，应返回明确错误或不确定性说明。
